# Proposal

## Why

The README should introduce Longgu to external readers as a usable project. It currently mixes product introduction with internal planning, OpenSpec workflow details, and version history. The project also needs an explicit non-commercial license instead of any MIT-style commercial permission.

## What Changes

- Rewrite README as a public-facing project introduction focused on current capabilities and usage.
- Move development workflow guidance out of README and keep it in AGENTS.md.
- Add an explicit non-commercial software license file.
- Mark package metadata with a non-commercial license identifier.

## Impact

- Documentation-only behavior change plus package metadata license update.
- No CLI behavior, schemas, or runtime code changes.
